Start Your Own Blog with Jornal

A step-by-step guide for designers and non-technical people to launch their own blog using GitHub Pages—no coding required.

You're reading this blog. You like how simple it is. You want one just like it. Good news: you can have it running in about 10 minutes, and you don't need to know how to code.

This guide will walk you through everything—from creating a GitHub account to publishing your first post. Let's do this.

What You'll Need

Before we start, here's what you'll need:

  • A GitHub account (free)
  • Claude Code or access to Claude
  • About 10 minutes of your time

That's it. No servers to configure. No databases to set up. No monthly hosting fees.

Step 1: Create a GitHub Account

If you don't have a GitHub account yet, head to github.com and sign up. It's free. Pick a username you like—it'll be part of your blog's URL.

Once you're in, you're ready for the next step.

Step 2: Fork the Jornal Repository

Here's where the magic happens. Go to the Jornal repository:

github.com/abduzeedo/jornal

Click the "Fork" button in the top right corner. This creates your own copy of the blog that you can customize however you want.

When prompted, you can rename it to whatever you like. If you want the cleanest URL possible, name it yourusername.github.io (replacing "yourusername" with your actual GitHub username).

Step 3: Enable GitHub Pages

Now let's make your blog live on the internet:

  1. Go to your forked repository
  2. Click "Settings" (the gear icon)
  3. Scroll down to "Pages" in the left sidebar
  4. Under "Source," select "Deploy from a branch"
  5. Choose "main" branch and "/ (root)" folder
  6. Click "Save"

Wait a minute or two, then refresh the page. You'll see a green box with your live URL. Click it. Your blog is now live on the internet.

Your blog URL will be something like: yourusername.github.io/jornal

Step 4: Clone It to Your Computer

To make changes and write posts, you'll want the files on your computer. The easiest way:

  1. Download and install GitHub Desktop (it's free and beginner-friendly)
  2. Sign in with your GitHub account
  3. Click "Clone a repository from the Internet"
  4. Find your forked Jornal repository and clone it
  5. Choose where to save it on your computer

Now you have all the blog files on your computer, ready to customize.

Step 5: Make It Yours

Open the folder in Claude Code (or your preferred editor with Claude). Now the fun begins.

Just tell Claude what you want to change:

  • "Change the site title to My Blog"
  • "Update the about page with my bio"
  • "Change the accent color to green"
  • "Add my email to the contact page"

Claude will make the changes for you. Review them, and when you're happy, move to the next step.

Step 6: Publish Your Changes

After making changes, you need to push them to GitHub. In GitHub Desktop:

  1. You'll see your changes listed on the left
  2. Write a short summary (like "Updated site title")
  3. Click "Commit to main"
  4. Click "Push origin"

Wait about a minute, then refresh your live site. Your changes are now live.

Step 7: Write Your First Post

This is the best part. Open the folder in Claude Code and say:

"Let's blog"

Then describe what you want to write about. Claude will:

  • Write the post based on your description
  • Create the HTML file
  • Update the blog listing
  • Update the home page

Review the post, make any tweaks you want, then commit and push. Your new post is live.

Tips for Success

A few things I've learned along the way:

  • Be specific. Instead of "make it look better," try "make the headings larger and add more space between paragraphs."
  • Preview locally. Run python3 -m http.server 8000 in your terminal, then open localhost:8000 to see changes before publishing.
  • Commit often. Small, frequent commits are easier to track than big changes.
  • Don't be afraid to experiment. You can always undo changes with Git.

What You Get

With this setup, you have:

  • Free hosting — GitHub Pages costs nothing
  • Fast loading — Static HTML is as fast as it gets
  • Full control — Every pixel is yours to customize
  • No maintenance — No plugins to update, no security patches
  • Version history — Git tracks every change you make
  • AI-powered writing — Claude helps you create content

Go Build Something

That's it. You now have everything you need to run your own blog. No excuses about not being technical enough. No waiting for the "right" platform. Just start writing.

The tools have never been more accessible. The only thing left is your ideas.
