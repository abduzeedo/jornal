Vibe Coding for Designers

January 26, 2026

I'm going to let you in on a secret: I have no idea what I'm doing. Well, okay, that's not entirely true. I know what I want to do. I just don't always know the exact syntax to make it happen. And apparently, in 2026, that's totally fine.

Welcome to vibe coding.

What Even Is Vibe Coding?

Vibe coding is when you describe what you want to build, and AI helps you write the code. You bring the vision, the taste, the "I want it to feel like this" energy. The AI handles the semicolons and the curly braces and remembering whether it's forEach or for...of (honestly, I still mix those up).

It's not cheating. It's collaboration. Like having a really patient senior developer sitting next to you who never sighs when you ask the same question twice.

Why This Matters for Designers

Here's the thing: designers have always been able to think in systems. We understand hierarchy, flow, state changes, user journeys. We've just been locked out of building because the syntax was a barrier.

Vibe coding removes that barrier. Not entirely—you still need to understand concepts, and you'll definitely break things. But the gap between "I can imagine this" and "I can ship this" just got a lot smaller.

The best code is the code that exists and works. Perfection is for version two.

How We Built This Blog

This site you're reading right now? It started with a simple conversation. I wanted a blog. No databases. No complicated build tools. No waiting for Webpack to do whatever Webpack does.

Just HTML files. Fast. Simple. Something I could actually understand when I looked at it six months from now.

So I described what I wanted: a home page, an about page, a contact page, and a blog section where I could add posts by just... asking for them. Each post becomes a new HTML file. The blog listing updates. Done.

The whole thing took one conversation. I said "let's blog," described what I wanted to write about, and here we are. You're reading the output.

It's Not Magic (But It Feels Like It)

Look, I'm not going to pretend I suddenly understand JavaScript. I don't. I still look at some code and my brain just plays elevator music. But I can read it now, kind of. I can point at things and say "that part handles the click" or "this is where it breaks."

And more importantly, I can build things. Real things. Things that work in a browser and do what I want them to do.

Is the code perfect? Absolutely not. Is it shipped? Yes. And shipped beats perfect every single time.

The Goal: Simple and Fast

This whole project is an experiment in minimalism. No frameworks. No dependencies. No build step. You open the HTML file, and it works. You want a new blog post, you ask for one. It appears.

That's it. That's the whole system.

Sometimes the best architecture is no architecture. Sometimes the best CMS is just... asking nicely.

If you're a designer who's ever felt intimidated by code, I get it. I've been there. I'm still there, honestly. But vibe coding has changed the game. You don't need to be a developer to build things anymore. You just need to know what you want and be willing to describe it.

The vibes will handle the rest.
